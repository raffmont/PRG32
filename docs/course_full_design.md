# PRG32 Full Course Design

## Course title

**Low-Level Game Systems with RISC-V and Embedded Platforms**

## Duration

- 12 weeks
- 2 lectures + 1 lab per week

## Learning outcomes

Students will:

- Understand assembly-level program structure
- Build interactive systems with a runtime loop
- Manage input/output in embedded systems
- Reason about memory layout and ABI design
- Deliver a complete working software artifact

## Weekly breakdown

### Week 1 — Introduction
- What is PRG32
- Toolchain overview
- QEMU vs hardware

### Week 2 — Runtime model
- Study `main.c`
- Game loop concept

### Week 3 — Assembly basics
- Registers, memory, control flow

### Week 4 — Cartridge structure
- `.prg32` format
- Entry points

### Week 5 — Input handling
- Controller abstraction

### Week 6 — Graphics basics
- Pixels, text

### Week 7 — Animation
- Frame updates

### Week 8 — Sprites
- Rendering and composition

### Week 9 — Collision
- Hitboxes and logic

### Week 10 — Systems thinking
- Performance
- Memory constraints

### Week 11 — Project work
- Game development

### Week 12 — Presentation
- Demo + technical discussion

## Assessment

- Labs: 40%
- Final project: 40%
- Oral/technical discussion: 20%

## Final project requirements

- Playable game
- Uses input, graphics, and game loop
- Packaged as `.prg32`
- Runs on QEMU

## Optional extensions

- Hardware deployment
- Network score submission
- Performance optimization

## Instructor notes

- Emphasize understanding over code quantity
- Encourage experimentation
- Use leaderboard to motivate engagement
